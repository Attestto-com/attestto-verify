Quick Start

Add document verification to your site in 30 seconds:

HTML 
<!-- Add to any HTML page -->
<script type="module" src="https://unpkg.com/@attestto/verify"></script>

<!-- Verify documents -->
<attestto-verify></attestto-verify>

<!-- Sign with DID wallet -->
<attestto-sign></attestto-sign>

That's it. Both components are self-contained with Shadow DOM — no CSS conflicts, no global state.

<attestto-verify>

Drop-in document verification. Extracts SHA-256 hash, detects PAdES digital signatures, runs forensic security audit. All client-side.

Attributes

AttributeTypeDefaultDescription
hash string — Pre-computed hash to verify against
allow-plugins boolean false Enable the plugin pipeline
theme "light" | "dark" "light" Color theme
root-ca string — Expected root CA name for trust validation

What It Does

1

Hash: Computes SHA-256 of the entire file

2

Signatures: Byte-scans for PAdES/PKCS#7 signature dictionaries

3

Forensic Audit: Checks for JavaScript, OpenAction, embedded files, ByteRange gaps, LTV data, encryption

4

Plugins: Runs registered verifier plugins (DID resolver, trust registry, etc.)

<attestto-sign>

Sign documents with any DID wallet. Uses @attestto/id-wallet-adapter for universal wallet discovery — any extension that calls registerWallet() will appear automatically.

Sign Flow

1

Discover: Finds installed credential wallets via discoverWallets()

2

Select: Auto-connects if one wallet, shows picker if multiple

3

Drop PDF: Computes SHA-256 hash of the document

4

Sign: Sends hash to wallet via requestSignature() — wallet shows consent popup

5

Store: Signed VC (DocumentSignatureCredential) is pushed to the wallet for storage

💡
Output is a Verifiable Credential, not a file download. The signed VC lives in the user's wallet and can be presented to any verifier. An "Export JSON" fallback is available.

Events

All events are composed — they cross Shadow DOM boundaries and bubble to your document root.

EventDetailWhen
verification-started { fileName, fileSize } User drops a file
verification-complete { hash, signatures, plugins } All checks finish
JavaScript — Listen for events 
const el = document.querySelector('attestto-verify')

el.addEventListener('verification-complete', (e) => {
  console.log('Hash:', e.detail.hash)
  console.log('Signatures:', e.detail.signatures.length)

  // Send hash to your backend for recording
  fetch('/api/verify', {
    method: 'POST',
    body: JSON.stringify(e.detail)
  })
})

Plugin System

Four plugin types run in a fixed "sandwich" order — parser and crypto run before display, trust and verifier run after. This ensures security checks can't be bypassed by a rogue plugin.

PDF File
→
Parser
→
Crypto
→
Trust
→
Verifier
→
Result
TypePurposeExample
parserExtract data from document formatPDF form fields, XML schema
cryptoCryptographic verificationpkijs, Ed25519, RSA
trustTrust anchor resolutionCA chain, AATL, EU TSL
verifierIdentity verificationDID resolver, vLEI, Solana
JavaScript — Register a plugin 
import { attesttoPlugins } from '@attestto/verify'

attesttoPlugins.register({
  type: 'verifier',
  name: 'my-did-resolver',
  label: 'My DID Resolver',

  async verify(result) {
    const did = result.signatures[0]?.signer.did
    if (!did) return { valid: false, message: 'No DID found' }

    const doc = await resolveDid(did)
    return {
      valid: doc.verified,
      message: `Resolved ${did}`,
      details: { document: doc }
    }
  }
})

// CDN users: window.Attestto.registerPlugin({ ... })

Result Schema

Every verification returns an AttesttoVerificationResult — the universal contract that bridges PDF/PAdES and DID/VC verification through a single interface.

TypeScript — AttesttoVerificationResult 
interface AttesttoVerificationResult {
  verified: boolean
  timestamp: string                // ISO 8601
  document: DocumentInfo           // file, hash, format
  integrity: IntegrityResult       // hash + ByteRange
  signatures: SignatureResult[]    // one per signer
  extensions: ExtensionResult[]    // plugin outputs
}

interface SignatureResult {
  level: VerificationLevel        // detected → qualified
  type: SignatureType             // pades, did-ecdsa, vc-jwt...
  signer: SignerIdentity          // name, org, DID, LEI
  trust: TrustInfo                // source, level, details
  metadata: Record<string, unknown>
}

type SignatureType =
  | 'pades' | 'cades' | 'pades-legacy'
  | 'did-ecdsa' | 'did-eddsa'
  | 'vc-jwt' | 'vc-ld' | 'sd-jwt'
  | 'xml-dsig' | 'unknown'

Trust Levels

Signatures progress through a trust ladder. Each level builds on the previous:

Detected Signature structure found (v1 byte scan)
Signed Cryptographic math verifies — key matches signature
Trusted Chain reaches a known root (CA or DID trust registry)
Qualified Meets regulatory standard (eIDAS qualified, gov-issued)
ℹ️
v1 reaches "detected" level. Cryptographic verification (signed → qualified) requires the v2 crypto plugin (ATT-209) which uses pkijs for full PKCS#7/CMS parsing.

Forensic Audit

Every PDF is scanned for security properties — no network requests, all done locally via byte scanning:

CheckWhatWhy
pdfVersionPDF specification versionCompatibility baseline
hasJavaScriptEmbedded JavaScriptSecurity risk — code execution
hasOpenActionAuto-execute on openPhishing / payload trigger
hasEmbeddedFilesAttached files inside PDFHidden payload delivery
byteRangesSignature byte rangesTamper detection — gaps = modified
hasLTVLong-Term Validation dataOffline revocation check capability
isEncryptedDocument encryptionAccess control present

Wallet Adapter

The sign component uses @attestto/id-wallet-adapter — a universal protocol for credential wallet discovery and signing. Any wallet extension can participate:

TypeScript — Extension side (register your wallet) 
import { registerWallet } from '@attestto/id-wallet-adapter'

registerWallet({
  did: 'did:web:yourwallet.com',
  name: 'Your Wallet',
  icon: 'https://yourwallet.com/icon.svg',
  version: '1.0.0',
  protocols: ['chapi'],
  maintainer: { name: 'Your Org' }
})
TypeScript — Site side (discover + sign) 
import {
  discoverWallets, requestSignature
} from '@attestto/id-wallet-adapter'

// Find installed wallets
const wallets = await discoverWallets()

// Request a signature
const result = await requestSignature(wallets[0], {
  hash: 'a1b2c3...',
  fileName: 'contract.pdf',
  hashAlgorithm: 'SHA-256'
})

if (result?.approved) {
  console.log('Signed by', result.did)
}
